### 정적

- 시간과 더불어 변화하지 않는 고정된 상태
- 프로그램의 실행에 들어가기 전에, 미리 결정할 수 있는 성질

### 동적

- 시시각각으로 변화하는 상태
- 프로그램의 실행 시점에 상황에 따라 결정할 수 있는 성질

### 정적과 동적

동적…?

- 동적 메모리
- 동적라이브러리 (DLL): 동적 연결
- 동적 바인딩
- 동적 우선순위 (스레드의 우선순위)

⇒ 결정하는 시점으로 구분한다!

### 결정 시점

프로그램(.exe) → 컴파일 시점(정적) → 실행 시점(동적) →  종료 시점

즉, 무언가를 결정하는 시점이 컴파일 시점 VS. 실행 시점

### 동적의 특징

- 동적은 정적에 비해 성능이 나쁘다
- 동적은 정적에 비해, 유연성이 좋다
- 실행시간(Run Time)에도 두 가지가 있음
    - 실행 시작
    - 실행 중 (실행 시점을 늦출수록 유연성이 좋아짐)
